
🧠 Executive Summary
Active exploitation remains concentrated on edge and client attack surfaces: Ivanti EPMM (CVE-2026-1340/1281) and an unpatched Adobe Reader zero-day campaign.
Taiwan-focused intrusions using new Lua malware (LucidRook) show mature tradecraft: spear-phishing, DLL sideloading, geo-fenced execution, FTP/Gmail-based exfiltration.
E-commerce skimming operations escalated with SVG-inline Magecart payloads and ongoing PolyShell abuse, impacting ~100 Magento stores in the latest wave.
Supply-chain compromise of Smart Slider 3 Pro update channel delivered multi-layer WordPress/Joomla backdoors and hidden admin creation.
Healthcare ransomware disruption (ChipSoft) indicates continued pressure on healthcare IT providers as high-leverage targets.
🚨 Active Threats & Vulnerabilities
CVE-2026-1340 / CVE-2026-1281 (Ivanti EPMM)

Severity: Critical

Affected: Ivanti Endpoint Manager Mobile appliances (internet-exposed/unpatched)

Exploitation: In-the-wild (zero-day abuse; CISA-directed remediation action reported Apr 8)

Root cause: Unauthenticated code injection leading to RCE

Adobe Reader zero-day (no CVE assigned publicly as of Apr 10, 2026)

Severity: High-Critical (practical impact)

Affected: Adobe Reader (latest tested build per researcher)

Exploitation: In-the-wild

Root cause: Abuse of privileged Acrobat APIs (util.readFileIntoStream, RSS.addFeed) enabling data theft + staged follow-on JS delivery (potential RCE/SBX chain)

PolyShell (Magento/Adobe Commerce unrestricted upload flaw; no production patch at time of latest campaign reporting)

Severity: Critical

Affected: Magento Open Source / Adobe Commerce v2 stable branches

Exploitation: In-the-wild, mass automation

Root cause: REST API file upload validation failures (option validation/type gating/extension controls)

Smart Slider 3 Pro malicious update (v3.5.1.35)

Severity: High

Affected: WordPress/Joomla sites using compromised Pro update

Exploitation: In-the-wild (supply-chain)

Root cause: Compromised update distribution path delivering embedded backdoors/credential theft logic

🦠 Malware & Campaign Analysis
Campaign: LucidRook / LucidPawn / LucidKnight

Threat actor: UAT-10362 (Talos tracking)

Initial access: Spear-phishing with password-protected archives containing LNK/EXE droppers

Execution chain: User opens lure archive → LNK/EXE launcher → DLL sideloading (DismCore.dll) via legitimate binaries → LucidRook loads Lua bytecode stage from C2

Persistence: Startup-folder LNK launching dropped masqueraded binary (msedge.exe)

C2: Plaintext FTP for staging/exfil (archive1.zip/archive4.zip), plus LucidKnight SMTP via Gmail

MITRE ATT&CK: T1566.001, T1204.002, T1574.001, T1547.001, T1027, T1071.002, T1041

Campaign: SVG Magecart skimmer wave (Magento)

Threat actor: Unknown (crimeware)

Initial access: Exploitation of PolyShell on vulnerable Magento stores

Execution chain: Polyglot upload/webshell → injected 1x1 SVG onload payload (atob + setTimeout) → fake checkout overlay → card harvest + XOR/base64 encoding → exfil to /fb_metrics.php

Persistence: Webshell placement (accesson.php) + malicious JS/localStorage marker (_mgx_cv)

C2: HTTPS exfil to attacker domains resolving to single host

MITRE ATT&CK: T1190, T1505.003, T1059.007, T1041

Campaign: Adobe Reader fingerprinting exploit

Threat actor: Unknown

Initial access: Malicious PDF lure

Execution chain: Open PDF → obfuscated JS runs → privileged API abuse reads local files/fingerprints host → callback to attacker server for staged payload

Persistence: Not observed today

C2: HTTP(S) using “Adobe Synchronizer” user-agent pattern; callback infra observed

MITRE ATT&CK: T1203, T1059.007, T1005, T1041

Patch/workaround guidance:
Ivanti EPMM: immediately apply vendor mitigations/hotfixes for CVE-2026-1340/1281; treat exposed nodes as potentially compromised.
Smart Slider: remove v3.5.1.35, restore from pre-compromise backup, rotate all creds/salts, reinstall trusted components.
Magento/Adobe Commerce: enforce deny rules on pub/media/custom_options, scan for accesson.php, and monitor guest-cart REST abuse until production patching is available.
Adobe Reader zero-day: restrict untrusted PDF opening paths and enforce network controls for suspicious callback indicators pending vendor patch.
📊 Trends & Insights
Attackers are increasingly chaining low-noise staging with delayed second-stage delivery (Lua bytecode, remote JS callbacks), reducing forensic recoverability.
Public/commodity infrastructure abuse is growing (compromised FTP, analytics-looking endpoints, widely available hosting), complicating static blocklist defense.
Web supply-chain and e-commerce app-layer exploitation remain high-ROI, with rapid operationalization from exploit to payment theft.
Sector focus remains on healthcare and high-value data brokers, with outages now used as immediate leverage alongside data theft.
